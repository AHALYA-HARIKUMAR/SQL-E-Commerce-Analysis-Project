# E-Commerce Customer & Sales Intelligence System

## Project Overview

This project focuses on designing and analyzing an E-Commerce database using SQL. The database was built to simulate real-world business operations and generate insights related to customers, products, sales, orders, inventory, and employee performance.

The project demonstrates database design, data management, and business analysis using SQL queries.

---

## Objectives

- Design a relational database for an E-Commerce business.
- Establish relationships using Primary Keys and Foreign Keys.
- Store and manage customer, product, order, review, and employee data.
- Perform business analysis to generate actionable insights.
- Apply SQL concepts used in real-world data analytics projects.

---

## Database Schema

The project consists of the following tables:

### Customers
Stores customer information including demographics and location details.

### Products
Contains product details such as category, brand, price, and stock quantity.

### Orders
Stores order transactions and payment information.

### Order_Items
Maintains product-level details for each order.

### Reviews
Contains customer ratings and product feedback.

### Employees
Stores employee information including department and salary details.

---

## Entity Relationships

- One Customer can place multiple Orders.
- One Order can contain multiple Products.
- One Product can appear in multiple Orders.
- Customers can provide Reviews for Products.
- Employees are maintained for organizational analysis.
